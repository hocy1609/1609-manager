# 16:09 Manager

Менеджер для **Neverwinter Nights: Enhanced Edition**. Помогает запускать игру с профилями, управлять серверами и следить за логами, чтобы упростить ежедневные игровые рутины.

---

## ✨ Возможности

### 🎮 Профили и быстрый запуск

| Функция | Описание |
|---------|----------|
| **Управление профилями** | Создание, редактирование, удаление и дублирование игровых профилей (аккаунт, CD-ключ, имя игрока, сервер) |
| **Групповой запуск** | Выделение нескольких профилей через `Ctrl`/`Shift` или **Middle Click**, запуск одной кнопкой |
| **Запуск по категории** | Запуск всех профилей одной категории нажатием ▶️ рядом с её названием |
| **Категории и сортировка** | Группировка профилей по категориям, переименование, сортировка ⬆/⬇, сворачивание/разворачивание |
| **Drag & Drop** | Перетаскивание профилей между категориями и изменение их порядка |
| **Автоматический Safe-Exit** | Корректное закрытие игры через автоматическую последовательность ESC → клик «Выход» → клик «Подтвердить» |
| **Перезапуск** | Перезапуск выбранных профилей: закрытие через Safe-Exit и повторный запуск |
| **Мониторинг процессов** | Автоматическое отслеживание PID запущенных сессий с индикацией статуса в списке |

### 🖥 Управление серверами

| Функция | Описание |
|---------|----------|
| **Список серверов** | Добавление, удаление, группировка серверов по группам (Siala и др.) |
| **Статус-проверка** | Пинг серверов с отображением задержки, статуса (`🟢 Online` / `🔴 Offline`), наличия пароля и количества игроков |
| **Автопинг** | Периодическая проверка всех серверов в фоне (по умолчанию каждые 5 мин) |
| **Автоподключение** | Привязка конкретного сервера к профилю — при запуске клиент подключается автоматически |

### 🔑 Менеджер CD-ключей

| Функция | Описание |
|---------|----------|
| **Хранение ключей** | Привязка CD-ключа к каждому профилю с валидацией формата (`XXXXX-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX-XXXXX`) |
| **Быстрое переключение** | Автоматическая замена ключа в `nwncdkey.ini` при запуске профиля |

### ⌨️ Горячие клавиши (Hotkeys)

| Функция | Описание |
|---------|----------|
| **Глобальные хоткеи** | Регистрация глобальных горячих клавиш через Windows `RegisterHotKey` API |
| **Редактор макросов** | Настройка триггера (напр. `ALT+1`) и последовательности нажатий (напр. `F1, SPACE, ENTER`) |
| **Правый клик** | Опциональная отправка правого клика перед последовательностью |
| **Master Toggle** | Глобальная горячая клавиша для включения/выключения всех хоткеев (по умолчанию `ALT+S`) |
| **Умная активация** | Хоткеи срабатывают только для профиля с включёнными хоткеями, чьё окно сейчас активно |
| **Focus-watcher** | Фоновый поток отслеживает активное окно и динамически регистрирует/снимает хоткеи |

### 📋 Мониторинг логов

| Функция | Описание |
|---------|----------|
| **Discord Webhooks** | Отслеживание ключевых слов в лог-файле и отправка уведомлений через Discord Webhook |
| **Open Wounds (Slayer)** | Автоматическое нажатие заданной F-клавиши при обнаружении срабатывания Open Wounds с кулдауном 2 сек |
| **Auto-Fog** | Автоматическая отправка консольной команды `##DebugMode 1` + `##dm_setvarint PERS_FOG 1` при входе в Full PVP-зону |
| **Счётчик попаданий** | Отображение счётчика Slayer-попаданий в UI |
| **Ожидание игры** | Мониторинг автоматически запускается при появлении процесса игры и останавливается при выходе |

### ⚙️ Автоматизация настроек NWN

| Функция | Описание |
|---------|----------|
| **Правка `settings.tml`** | Автоматическое обновление имени игрока (`PlayerName`) в `settings.tml` при переключении профиля |
| **Правка `nwncdkey.ini`** | Подстановка CD-ключа перед каждым запуском |
| **Авто-определение путей** | Поиск `nwmain.exe` через реестр Steam/GOG |
| **DPI Awareness** | Автоматическая настройка Per-Monitor DPI Awareness v2 для корректного отображения на HiDPI-мониторах |

### 💾 Бэкапы и экспорт

| Функция | Описание |
|---------|----------|
| **Автобэкап** | Автоматическое резервирование `nwn_settings.json` при запуске |
| **Ротация бэкапов** | Хранение до 10 последних резервных копий с автоматической очисткой старых |
| **Экспорт** | Сохранение всех данных (профили, серверы, хоткеи, настройки) в JSON-файл |
| **Селективное восстановление** | Импорт данных с выбором категорий: профили, серверы, хоткеи, настройки лог-монитора, общие настройки |
| **Слияние профилей** | При импорте — добавление новых профилей без удаления существующих (по совпадению `playerName` + `cdKey`) |
| **Импорт xNwN.ini** | Импорт аккаунтов, ключей и серверов из конфигурации xNwN |
| **accounts.txt** | Экспорт/импорт профилей в простом текстовом формате |

### 🎨 Интерфейс и темы

| Функция | Описание |
|---------|----------|
| **6 цветовых тем** | `dark`, и другие — мгновенное переключение с полной перестройкой UI |
| **Кастомный title-bar** | Собственная панель заголовка с перетаскиванием, минимизацией и закрытием |
| **Сворачивание в трей** | Минимизация в системный трей через `pystray` с меню «Show» / «Quit» |
| **Адаптивная раскладка** | Автоматическая подстройка отступов в зависимости от ширины окна |
| **Inline-кнопки** | Кнопки «Редактировать» / «Удалить» появляются при наведении на профиль |
| **Status-bar** | Панель состояния: количество сессий, статус хоткеев, статус лог-монитора |
| **Тултипы** | Всплывающие подсказки на элементах управления (отключаемые) |
| **Кастомные шрифты** | Шрифт Mona Sans загружается из папки `fonts/` |

### 🖥 Экраны приложения

| Экран | Описание |
|-------|----------|
| **Home** | Главный экран: список профилей, панель информации, серверы, кнопки запуска |
| **Hotkeys** | Конфигурация макросов: добавление, редактирование, удаление биндов, master toggle |
| **Log Monitor** | Настройка мониторинга логов: вебхуки, ключевые слова, Open Wounds, Auto-Fog |
| **Settings** | Пути, координаты выхода, тема, тултипы, экспорт/импорт, бэкапы |

```

│   ├── server_manager.py   — CRUD серверов, пинг, авто-пинг
│   ├── keybind_manager.py  — Глобальные хоткеи через RegisterHotKey, focus-watcher
│   ├── log_monitor_manager.py — Управление мониторингом логов, Open Wounds, Auto-Fog
│   ├── data_manager.py     — Загрузка/сохранение/экспорт/импорт данных
│   ├── settings_manager.py — Логика диалога настроек
│   ├── theme_manager.py    — Применение тем через полную перестройку UI
│   ├── tray_manager.py     — Управление системным треем (pystray)
│   ├── ui_state.py         — Состояние UI, навигация между экранами
│   └── error_handler.py    — Централизованная обработка ошибок
│
├── ui/                     — UI-слой (Tkinter)
│   ├── ui_base.py          — Палитра цветов, базовые виджеты, COLORS, ModernButton
│   ├── components.py       — TitleBar, StatusBar, NavigationBar
│   ├── screens/            — Экраны: Home, Hotkeys, Log Monitor, Settings, Help
│   └── dialogs/            — Модальные диалоги: профили, серверы, ключи, бэкапы, настройки
│
├── utils/                  — Утилиты
│   ├── win_automation.py   — Windows API: DPI, SendInput, safe_exit, settings.tml, авто-определение путей
│   └── log_monitor.py      — LogMonitor: tail лог-файла, фильтр по ключевым словам
│
├── assets/                 — Иконки и ресурсы
├── fonts/                  — Шрифт Mona Sans
└── macros/                 — Шаблоны макросов
```

## 🧰 Требования

- **Windows** (используются Windows API и пути по умолчанию)
- **Python 3.11+**

Зависимости:

| Пакет | Назначение |
|-------|-----------|
| `pystray` ≥ 0.19.5 | Системный трей |
| `Pillow` ≥ 10.0.0 | Обработка изображений (иконка трея) |
| `pyinstaller` ≥ 6.0.0 | Сборка `.exe` (опционально) |

```bash
pip install -r requirements.txt
```

---

## 🚀 Запуск

```bash
python app.py
```

Все пользовательские данные и логи сохраняются в папке **`1609 settings`** рядом с исполняемым файлом/проектом.

---

## 📦 Сборка (Windows)

Для сборки standalone `.exe` используется **PyInstaller** и скрипт `build.bat`:

```bat
build.bat
```

Готовый файл появится в `dist/1609Manager.exe`.
